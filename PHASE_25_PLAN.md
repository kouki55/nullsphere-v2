# Phase 25: 脅威データの永続化と高度な分析機能の強化 実装計画

## 1. 目的
Phase 24 で実現したリアルタイム脅威フィードを拡張し、受信したデータをデータベースに永続化するとともに、蓄積されたデータに基づいた高度な分析・可視化機能を提供します。

## 2. 実装項目

### 2.1 データベーススキーマの拡張
- `threat_feeds` テーブルの追加: リアルタイムフィードの内容を保存。
- `threat_analytics` ビュー/テーブルの検討: 統計情報の高速取得用。

### 2.2 バックエンド (Node.js / Drizzle ORM)
- **永続化ロジック**: `KernelBridge` で受信したイベントを `threat_feeds` テーブルに非同期で保存。
- **分析 API (tRPC)**:
    - 時系列データ取得 (脅威発生数の推移)
    - 深刻度別分布
    - 攻撃元 IP ランキング
    - 攻撃タイプ別統計

### 2.3 フロントエンド (React / Recharts)
- **分析ダッシュボード**:
    - 脅威トレンドチャート (Line Chart)
    - 深刻度分布 (Pie Chart)
    - トップアタッカーリスト
- **フィルタリング機能**: 期間指定、深刻度フィルタ。

## 3. スケジュール
1. スキーマ設計とマイグレーション
2. 永続化ロジックの実装
3. 分析 API の実装
4. フロントエンド UI の構築
5. 統合テストとデプロイ
