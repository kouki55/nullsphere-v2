## 概要

**NullSphere V2** は、CISO・SOC（セキュリティオペレーションセンター）向けの統合セキュリティ管理ダッシュボードです。Linux カーネルレベルのセキュリティイベント監視、ウィルス隔離、脅威インテリジェンス、リアルタイム通知を一元管理できます。

### 主な特徴

NullSphere V2 は以下の4つのコアコンポーネントで構成されています。

| コンポーネント | 機能 | 説明 |
|---|---|---|
| **NullSphere Engine** | eBPF カーネルフック | Linux カーネルレベルでプロセス・ファイル・ネットワークアクティビティを監視 |
| **The Void** | Micro-VM 隔離 | 検知された脅威を自動的に隔離 VM 環境に転送 |
| **NullHorizon** | 攻撃者追跡 | 攻撃元の地理的位置特定・攻撃経路の可視化 |
| **Control Node** | 統合管理 | ダッシュボード・API・通知の中央管理 |

### 対応環境

- **OS**: Linux（Ubuntu 20.04 以上推奨）
- **ブラウザ**: Chrome、Firefox、Safari、Edge（最新版）
- **ネットワーク**: インターネット接続（Google Maps API 使用）

| CPU | 2 コア | 4 コア以上 |
| メモリ | 4 GB | 8 GB 以上 |
| ディスク | 20 GB | 100 GB 以上 |

- Node.js 18.0 以上
- Python 3.8 以上
- MySQL 5.7 以上（または互換 DB）
- Linux カーネル 5.8 以上（eBPF サポート）

## インストール・セットアップ

### 1. 前提条件の確認

```bash
# Node.js バージョン確認
node --version

# Python バージョン確認
python3 --version

# Linux カーネルバージョン確認
uname -r
```

### 2. リポジトリのクローン

```bash
git clone https://github.com/your-org/nullsphere-v2.git
cd nullsphere-v2
```

### 3. 依存パッケージのインストール

```bash
# Node.js パッケージ
pnpm install

# Python パッケージ
pip3 install -r requirements.txt
```

### 4. 環境変数の設定

`.env.local` ファイルを作成し、以下の環境変数を設定してください。

```bash
# データベース
DATABASE_URL=mysql://user:password@localhost:3306/nullsphere

# OAuth
VITE_APP_ID=your-app-id
OAUTH_SERVER_URL=https://api.manus.im
VITE_OAUTH_PORTAL_URL=https://manus.im

# Google Maps API
VITE_GOOGLE_MAPS_API_KEY=your-api-key

# LLM API
BUILT_IN_FORGE_API_URL=https://api.manus.im
BUILT_IN_FORGE_API_KEY=your-api-key
```

### 5. データベースの初期化

```bash
# マイグレーション実行
pnpm run db:push

# シードデータ投入
node seed-db.mjs
```

### 6. 開発サーバーの起動

```bash
# 開発モード
pnpm run dev

# ブラウザで http://localhost:3000 にアクセス
```

### 7. 本番環境へのデプロイ

```bash
# ビルド
pnpm run build

# サーバー起動
pnpm run start
```

## 各機能の詳細ガイド

### 1. Dashboard（ダッシュボード）

**概要**: リアルタイムの脅威統計とシステム状態を一目で把握できるメインページです。

**主要要素**:

- **脅威統計カード**: アクティブな脅威数、ブロック済み脅威、隔離 VM 数などを表示
- **システムコンポーネント**: NullSphere Engine、The Void、NullHorizon、Control Node の稼働状況
- **Recent Threats**: 最新の脅威検知イベント一覧
- **Event Stream**: リアルタイムで流れるセキュリティイベント

**操作方法**:

1. ダッシュボードにアクセスすると、自動的に最新データが読み込まれます
2. 統計カードをクリックすると、詳細ページに遷移します
3. Event Stream のイベントをクリックすると、詳細情報が表示されます

### 2. Architecture（システムアーキテクチャビュー）

**概要**: NullSphere システムの4つのコアコンポーネントの状態を監視します。

**表示内容**:

各コンポーネントについて、以下の情報が表示されます。

| 情報 | 説明 |
|---|---|
| ステータス | OPERATIONAL / WARNING / ERROR |
| CPU 使用率 | リアルタイムの CPU 使用率（%） |
| メモリ使用率 | リアルタイムのメモリ使用率（%） |
| 処理スループット | 1 秒あたりのイベント処理数 |
| 稼働時間 | コンポーネント起動からの経過時間 |

**操作方法**:

1. 各コンポーネントをクリックすると、詳細ページに遷移します
2. グラフをマウスオーバーすると、時系列データが表示されます
3. 「詳細」ボタンをクリックすると、ログ・設定が表示されます

### 3. Data Flow（データフローシミュレーター）

**概要**: セキュリティイベント処理の6段階プロセスをアニメーション表示します。

**6段階プロセス**:

1. **検知（Detection）**: eBPF フックでイベント検知
2. **遮断（Blocking）**: 脅威を即座に遮断
3. **転送（Transfer）**: 隔離 VM 環境に転送
4. **欺瞞（Deception）**: デコイデータで攻撃者を欺瞞
5. **追跡（Tracking）**: 攻撃者の行動を追跡
6. **可視化（Visualization）**: ダッシュボードに表示

**操作方法**:

1. 「シミュレーション開始」ボタンをクリックすると、アニメーションが再生されます
2. 各段階をクリックすると、詳細説明が表示されます
3. 「速度調整」スライダーでアニメーション速度を変更できます

### 4. Threat Map（脅威マップ）

**概要**: Google Maps 上に攻撃元・攻撃経路・標的サーバーをマッピング表示します。

**表示要素**:

- **攻撃元マーカー**: 赤色ピン（攻撃者の位置）
- **経路線**: 青色の曲線（攻撃経路）
- **標的マーカー**: 緑色ピン（自社サーバー）
- **脅威レベル**: マーカーのサイズで脅威レベルを表示

**操作方法**:

1. マップをドラッグして移動、スクロールでズーム
2. マーカーをクリックすると、攻撃者情報が表示されます
3. 「フィルター」ボタンで表示対象を絞り込めます
4. 「エクスポート」ボタンで地図画像を保存できます

### 5. Attackers（攻撃者プロファイリング）

**概要**: 検知された攻撃者の詳細情報を表示・管理します。

**表示情報**:

| 項目 | 説明 |
|---|---|
| IP アドレス | 攻撃元 IP |
| 国・地域 | 地理的位置 |
| OS | 攻撃者マシンの OS |
| 脅威レベル | LOW / MED / HIGH / CRITICAL |
| 実行コマンド | 攻撃者が実行したコマンド履歴 |
| 初回検知 | 最初に検知された日時 |
| 最終活動 | 最後に検知された日時 |

**操作方法**:

1. 攻撃者一覧から対象を選択
2. 「詳細」ボタンをクリックすると、詳細プロファイルが表示されます
3. 「ホワイトリスト追加」でホワイトリストに登録
4. 「ブロック」で IP をブロック

### 6. Event Log（イベントログビューア）

**概要**: eBPF フック、VM 転送、デコイアクセスなどのセキュリティイベントをリアルタイム表示します。

**イベントタイプ**:

| タイプ | 説明 |
|---|---|
| exec | プロセス実行 |
| file | ファイルアクセス |
| net | ネットワーク通信 |
| kill | プロセス終了 |
| ptrace | プロセストレース |
| module_load | カーネルモジュールロード |

**操作方法**:

1. イベント一覧から対象を選択
2. 「詳細」をクリックすると、完全なイベント情報が表示されます
3. 「フィルター」で表示対象を絞り込み
4. 「エクスポート」で CSV 形式でダウンロード

### 7. VM Management（隔離 VM 管理）

**概要**: 隔離用 Micro-VM の起動・停止・リソース管理を行います。

**操作可能な項目**:

- **起動**: 新しい隔離 VM を起動
- **停止**: 実行中の VM を停止
- **リソース割り当て**: CPU・メモリ・ディスク容量を設定
- **スナップショット**: VM の状態を保存

**操作方法**:

1. VM 一覧から対象を選択
2. 「起動」「停止」ボタンで制御
3. 「リソース設定」で CPU・メモリを調整
4. 「スナップショット作成」で状態を保存

### 8. Decoy Control（デコイ生成）

**概要**: 偽パスワードファイル、囮 DB などの欺瞞データを生成・管理します。

**デコイタイプ**:

| タイプ | 説明 |
|---|---|
| Password File | 偽パスワードファイル（/etc/passwd など） |
| Database | 囮データベース |
| API Key | 偽 API キー |
| SSH Key | 偽 SSH キー |

**操作方法**:

1. 「新規作成」ボタンをクリック
2. デコイタイプを選択
3. パラメータを設定（ファイルパス、内容など）
4. 「生成」ボタンで作成
5. 「監視」ボタンでアクセスを監視

### 9. Notifications（通知）

**概要**: 重大脅威検知時の通知を一元管理します。

**通知チャネル**:

- **ブラウザ通知**: ダッシュボード内のポップアップ
- **メール**: 登録メールアドレスへの送信
- **Slack**: Slack ワークスペースへの投稿
- **SMS**: 携帯電話への SMS 送信

**操作方法**:

1. 通知一覧から対象を選択
2. 「既読」ボタンでマーク
3. 「削除」で削除
4. 「設定」で通知ルールを変更

### 10. AI Analysis（LLM 攻撃分析）

**概要**: LLM を活用して攻撃パターンを分析し、攻撃者の意図・次の行動予測・対策推奨を生成します。

**分析内容**:

- **攻撃パターン分析**: 検知されたイベントから攻撃の意図を推定
- **行動予測**: 攻撃者の次のステップを予測
- **対策推奨**: 推奨される防御策・対応手順

**操作方法**:

1. 分析対象のイベント・攻撃者を選択
2. 「分析開始」ボタンをクリック
3. LLM が分析を実行（数秒～数十秒）
4. 分析結果がレポート形式で表示されます
5. 「レポート保存」でダウンロード

## トラブルシューティング

### ダッシュボードが表示されない

**症状**: ブラウザで http://localhost:3000 にアクセスしても画面が表示されない

**原因と対応**:

1. **開発サーバーが起動していない**
   ```bash
   # サーバーの状態確認
   ps aux | grep "node"
   
   # サーバーの起動
   pnpm run dev
   ```

2. **ポート 3000 が使用中**
   ```bash
   # ポート確認
   lsof -i :3000
   
   # 別のポートで起動
   PORT=3001 pnpm run dev
   ```

3. **ブラウザキャッシュの問題**
   - ブラウザの開発者ツール（F12）を開く
   - キャッシュを削除（Ctrl+Shift+Delete）
   - ページを再読み込み（Ctrl+F5）

### イベントが表示されない

**症状**: Event Log に何もイベントが表示されない

**原因と対応**:

1. **カーネルモジュールがロードされていない**
   ```bash
   # モジュール確認
   lsmod | grep nullsphere
   
   # モジュールのロード
   sudo insmod nullsphere.ko
   ```

2. **nl_bridge が起動していない**
   ```bash
   # プロセス確認
   ps aux | grep nl_bridge
   
   # nl_bridge の起動
   sudo python3 nl_bridge_enhanced.py
   ```

3. **データベース接続エラー**
   ```bash
   # DB 接続確認
   mysql -u user -p -h localhost -D nullsphere
   
   # ログ確認
   tail -f /var/log/nullsphere_nl_bridge.log
   ```

### Google Maps が表示されない

**症状**: Threat Map ページで地図が表示されない

**原因と対応**:

1. **Google Maps API キーが未設定**
   ```bash
   # .env.local を確認
   grep GOOGLE_MAPS_API_KEY .env.local
   
   # API キーを設定
   echo "VITE_GOOGLE_MAPS_API_KEY=your-key" >> .env.local
   ```

2. **API キーの権限不足**
   - Google Cloud Console にアクセス
   - Maps JavaScript API を有効化
   - API キーの制限を確認

3. **ネットワーク接続エラー**
   ```bash
   # インターネット接続確認
   ping 8.8.8.8
   
   # DNS 確認
   nslookup maps.googleapis.com
   ```

### パフォーマンスが低下している

**症状**: ダッシュボードの動作が遅い

**原因と対応**:

1. **イベント数が多すぎる**
   ```bash
   # イベント数確認
   SELECT COUNT(*) FROM events;
   
   # 古いイベントを削除
   DELETE FROM events WHERE ts < DATE_SUB(NOW(), INTERVAL 30 DAY);
   ```

2. **メモリ不足**
   ```bash
   # メモリ使用状況確認
   free -h
   
   # Node.js メモリ制限を増加
   NODE_OPTIONS="--max-old-space-size=4096" pnpm run dev
   ```

3. **データベーススロー**
   ```bash
   # DB スロークエリログ確認
   tail -f /var/log/mysql/slow.log
   
   # インデックス作成
   CREATE INDEX idx_events_ts ON events(ts);
   ```

## FAQ

### Q1: NullSphere V2 はどのような脅威に対応していますか？

**A**: NullSphere V2 は以下の脅威に対応しています。

- マルウェア・ランサムウェア
- 権限昇格攻撃
- データ流出・盗聴
- 暗号化通信の悪用
- カーネルエクスプロイト
- ゼロデイ攻撃

ただし、NullSphere V2 自体は**検知・隔離・分析**を行うシステムであり、実際の脅威排除にはセキュリティチームの対応が必要です。

### Q2: イベントはどのくらいの期間保持されますか？

**A**: デフォルトでは、イベントは 30 日間保持されます。設定で変更可能です。

```bash
# .env.local で設定
EVENT_RETENTION_DAYS=90
```

### Q3: 複数のサーバーを監視できますか？

**A**: はい。複数のサーバーに nl_bridge をインストールし、各サーバーを NullSphere V2 に登録することで、一元管理できます。

### Q4: オフライン環境で使用できますか？

**A**: Google Maps 機能を除き、ほぼ全機能がオフラインで動作します。ただし、LLM 分析機能はインターネット接続が必須です。

### Q5: ダッシュボードのバックアップ方法は？

**A**: データベースをバックアップしてください。

```bash
# MySQL バックアップ
mysqldump -u user -p nullsphere > nullsphere_backup.sql

# リストア
mysql -u user -p nullsphere < nullsphere_backup.sql
```

### Q6: ユーザーアカウントを追加できますか？

**A**: 現在のバージョンでは、Manus OAuth を使用した認証のみサポートしています。複数ユーザーは OAuth ポータルで登録してください。

### Q7: ダッシュボードのカスタマイズはできますか？

**A**: ウィジェットの配置変更、カラースキーム変更などは、ソースコード編集で可能です。詳細は開発者向けドキュメントを参照してください。

### Q8: サポートはありますか？

**A**: 本ドキュメント、トラブルシューティングセクション、GitHub Issues で対応しています。商用サポートについては、営業にお問い合わせください。
